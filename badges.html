const StatusBadge = ({ variant, icon, children }) => {
  const variants = {
    popular: 'bg-red-50 text-red-800',
    new: 'bg-sky-100 text-sky-800',

  return (
    <span className={`inline-flex items-center gap-1 ${variants[variant]} text-xs font-semibold px-3 py-1 rounded uppercase tracking-wider`}>
      {icon && (
        <span className="inline-flex items-center justify-center w-3.5 h-3.5 text-[10px] bg-black/10 rounded-sm border border-current">
          {icon}
        </span>
      )}
      {children}
    </span>
  );
};

Usage Guidelines

‚úÖ Do

  • Use status badges sparingly - maximum 1-2 per product card
  • Always provide icon space for certification logos
  • Use uppercase text for status badges (POPULAR, NEW, LIMITED)
  • Use title case for order status badges (Quote Received)
  • Group certification badges together
  • Ensure sufficient color contrast for accessibility

‚ùå Don't

  • Don't use more than 2 status badges per product
  • Don't use certification badges without proper licensing
  • Don't make badges interactive unless they filter/navigate
  • Don't use custom colors outside the defined palette
  • Don't stretch badges - let content define width
  • Don't use badges for general text labels

Accessibility

‚úì

Color Contrast: All badge combinations meet WCAG AA standards (4.5:1 minimum)

‚úì

Icon Alt Text: Provide meaningful alt text for certification logo images

‚úì

Semantic HTML: Use <span> for non-interactive badges, <button> if clickable

‚úì
