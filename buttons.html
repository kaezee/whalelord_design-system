Buttons

Primary interaction elements with complete state specifications. Available in three sizes and two variants.

Filled Buttons

Primary action buttons with solid backgrounds. Use for main CTAs.

HTML 
<!-- Small -->
<button class="btn btn-sm btn-filled">Button</button>

<!-- Medium (Default) -->
<button class="btn btn-md btn-filled">Button</button>

<!-- Large -->
<button class="btn btn-lg btn-filled">Button</button>

Specifications

Size Height Padding Font Size
Small 32px 12px 16px 14px
Medium 40px 16px 24px 16px
Large 48px 20px 32px 18px

Outline Buttons

Secondary action buttons with transparent backgrounds. Use for secondary CTAs or alongside filled buttons.

HTML 
<button class="btn btn-md btn-outline">Button</button>

}

Fluid Width Buttons

Buttons that expand to fill their container. Useful for mobile layouts and form submissions.

HTML 
<button class="btn btn-md btn-filled btn-fluid">Continue</button>

Usage Guidelines

✅ Do

  • Use filled buttons for primary actions
  • Use outline buttons for secondary actions
  • Maintain minimum 44px touch targets on mobile
  • Use white buttons on blue backgrounds (#025E96)
  • Use blue buttons on white backgrounds
  • Maximum one primary (filled) button per section

❌ Don't

  • Don't use multiple filled buttons side-by-side
  • Don't use buttons smaller than 32px height
  • Don't override border-radius (keep 8px)
  • Don't use orange/yellow accent colors on buttons
  • Don't use outline buttons on colored backgrounds
  • Don't stretch buttons unnecessarily wide

Accessibility

✓

Color Contrast: All button text meets WCAG AA standards (4.5:1 minimum)

✓

Touch Targets: Minimum 44x44px on mobile devices

✓

Keyboard Navigation: All buttons focusable and activatable via keyboard

✓

Focus Indicators: Clear visual focus states for keyboard users

✓

Disabled State: Visually distinct and not keyboard-focusable
