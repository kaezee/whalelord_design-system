Buttons

Action elements that guide buyers through discovery, quoting, and ordering flows.

Usage & Purpose

Primary Buttons

Main call-to-action (Request Quote, Add to Cart, Submit Order). Use sparingly - one primary per screen section.

Outline Buttons

Secondary actions (View Details, Compare, Save for Later). Use alongside primary buttons or as standalone options.

Neutral Buttons

Tertiary actions (Cancel, Back, Close). Low emphasis for reversible or less important actions.

Hover States: All interactive buttons use Shore Hover (#D4EAF4) for light backgrounds. Borders are 1px for refined professional appearance.

Button Sizes

Three sizes optimized for desktop and mobile marketplace interfaces.

<button class="btn btn-sm btn-primary">Small</button>
<button class="btn btn-md btn-primary">Medium</button>
<button class="btn btn-lg btn-primary">Large</button>
<button className="bg-waves-500 hover:bg-waves-700 text-white px-4 h-8 rounded-lg font-medium">Small</button>

<button className="bg-waves-500 hover:bg-waves-700 text-white px-8 h-12 rounded-lg font-medium">Large</button>

Primary

Main call-to-action for critical marketplace actions: Request Quote, Place Order, Confirm Purchase. The Waves blue builds buyer confidence.

Primary Outline

Secondary CTAs with Waves emphasis: Compare Products, View Details, Save to List. Hovers to Shore Ice for clear feedback.

<button class="btn btn-md btn-primary-outline">View Details</button>

<button className="border border-waves-500 hover:bg-shore-ice hover:border-waves-700 text-pearl-800 px-6 h-10 rounded-lg font-medium transition-all hover:underline">View Details</button>

Secondary Outline

Tertiary actions with minimal emphasis: Cancel, Go Back, Skip Step. Neutral gray maintains focus on primary flows.

<button class="btn btn-md btn-secondary-outline">Cancel</button>

Fluid / Responsive Buttons

Full-width buttons for mobile checkout flows, form submissions, and stacked action layouts. Add btn-fluid class to any button variant.
