Buttons

Primary interaction elements with complete state specifications. Available in three sizes and three variants.

Primary Blue Buttons

Main call-to-action buttons. Use on light backgrounds for primary actions.

HTML 
<!-- Small -->
<button class="btn btn-sm btn-primary">Button</button>

<!-- Medium (Default) -->
<button class="btn btn-md btn-primary">Button</button>

<!-- Large -->
<button class="btn btn-lg btn-primary">Button</button>

Specifications

State Background Text Color Notes
Default #025E96 #FFFFFF Primary brand color
Hover #01385A #FFFFFF + text underline
Disabled #CBD5E0 #A0AEC0 cursor: not-allowed

Light Fluid Buttons

For use on dark backgrounds (#025E96 or darker). Available in fill and outline variants.

HTML 
<!-- Fill variant -->
<button class="btn btn-md btn-light-fill">Button</button>

<!-- Outline variant -->
<button class="btn btn-md btn-light-outline">Button</button>

Fill Variant Specifications

State Background Text Color Notes
Default #E6F2F8 #1A202C Light fill
Hover #CCE5F1 #1A202C + text underline

Outline Variant Specifications

State Border Text Color Background
Default 1px #E6F2F8 #E6F2F8 transparent
Hover 1px #CCE5F1 #E6F2F8 rgba(230, 242, 248, 0.1) + underline

Tertiary Stroke Buttons

Secondary action buttons. Use for less important actions alongside primary buttons.

HTML 
<button class="btn btn-md btn-tertiary">Button</button>

Specifications

State Border Text Color Background
Default 1px #A0AEC0 #1A202C transparent
Hover 1px #718096 #1A202C #EDF2F7 + underline
Disabled 1px #CBD5E0 #A0AEC0 transparent

Fluid Width Buttons

Buttons that expand to fill their container. Useful for mobile layouts and form submissions.

HTML 
<button class="btn btn-md btn-primary btn-fluid">Continue</button>
<button class="btn btn-md btn-tertiary btn-fluid">Cancel</button>

Usage Guidelines

✅ Do

  • Use primary blue (#025E96) buttons for main CTAs on light backgrounds
  • Use light fluid buttons (#E6F2F8) on dark blue backgrounds
  • Use tertiary stroke buttons for secondary actions
  • Maintain minimum 44px touch targets on mobile
  • Maximum one primary button per section
  • All buttons show text underline on hover

❌ Don't

  • Don't use multiple primary buttons side-by-side
  • Don't use buttons smaller than 32px height
  • Don't override border-radius (keep 8px)
  • Don't use light buttons on light backgrounds
  • Don't use primary blue buttons on dark backgrounds
  • Don't stretch buttons unnecessarily wide

Accessibility

✓

Color Contrast: All button text meets WCAG AA standards (4.5:1 minimum)

✓

Touch Targets: Minimum 44x44px on mobile devices

✓

Keyboard Navigation: All buttons focusable and activatable via keyboard

✓

Focus Indicators: Clear visual focus states for keyboard users

✓

Disabled State: Visually distinct and not keyboard-focusable

✓

Hover Feedback: Text underline appears on all button hovers for clarity
