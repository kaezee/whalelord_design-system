Buttons — Hybrid System v2.0

Complete button hierarchy with Ice hover states, surface-aware variants, and dark background support.

One-Line Principle: Surfaces carry tone. Buttons carry intent. Color carries meaning.

Light Backgrounds

Use these button variants on white, light gray, or Ice/Sand surfaces.

Primary (Filled Blue)

Main CTAs — Use for primary actions like "Submit", "Continue", "Buy Now"

<button class="btn btn-md btn-primary">Primary Button</button>

Primary Outline

Secondary CTAs with blue emphasis — Hovers to Ice surface (#D9F0F2)

<button class="btn btn-md btn-primary-outline">Primary Outline</button>

Secondary Outline

Tertiary actions with minimal emphasis — Gray border, subtle hover

<button class="btn btn-md btn-secondary-outline">Secondary Outline</button>

Neutral (White)

Minimal emphasis actions — Use for less important actions

<button class="btn btn-md btn-neutral">Neutral Button</button>

Dark Backgrounds

Use these button variants on primary blue (#025E96) or darker backgrounds.

Primary Light (Filled)

Light buttons on dark — Kelp 50 fill (#E6F2F8)

<button class="btn btn-md btn-primary-light">Primary Light</button>

Primary Light Outline

Light outline on dark — Subtle contrast with rgba hover

<button class="btn btn-md btn-primary-light-outline">Primary Light Outline</button>

Tertiary (Text Only)

Text-only buttons for dark backgrounds — Minimal visual weight

<button class="btn btn-md btn-tertiary-text">Tertiary Text</button>

Button Sizes

Size Class Height Padding (H) Font Size
Small btn-sm 32px 16px 14px
Medium (default) btn-md 40px 24px 16px
Large btn-lg 48px 32px 18px

Note: Outline buttons reduce horizontal padding by 1px to account for border width.

Usage Guidelines

✓ Do

  • Use Primary for main actions
  • Use Primary Outline for secondary CTAs with emphasis
  • Use Ice hover (#D9F0F2) for outline buttons
  • Maintain visual hierarchy (filled > outline > text)
  • Test on actual background colors

✗ Don't

  • Don't use more than one Primary button in a view
  • Don't use light buttons on light backgrounds
  • Don't create custom button colors outside the system
  • Don't skip disabled states
  • Don't use buttons for navigation (use links)
