Usage Guidelines

üé® Primary Actions

  • Primary 500 for main CTAs
  • Primary 700 for hover states
  • Primary 50/100 for light backgrounds
  • Never use primary colors for text alone

üìù Text Colors

  • Neutral 900 for body text
  • Neutral 600/700 for secondary text
  • Neutral 400/500 for disabled text
  • White (#FFFFFF) on dark backgrounds only

üî≤ Backgrounds

  • Neutral 50 for page backgrounds
  • White (#FFFFFF) for cards and panels
  • Primary 50 for subtle highlights
  • Primary 500 for hero sections

üìê Borders & Dividers

  • Neutral 200 for standard borders
  • Neutral 300 for stronger dividers
  • Primary 200 for active/hover borders
  • Keep borders subtle and minimal

Do's and Don'ts

‚úÖ Do

  • Use Primary 500 (#025E96) as main brand color
  • Maintain high contrast for text (4.5:1 minimum)
  • Use semantic colors for their intended purpose
  • Test color combinations with accessibility tools
  • Use neutral grays for most UI elements

‚ùå Don't

  • Use light blue (#E6F2F8) on white backgrounds
  • Mix semantic colors (red + green together)
  • Use color as the only indicator of state
  • Override primary brand color with custom blues
  • Use pure black (#000000) for text

Implementation

Use CSS custom properties for consistent color application:

/* Primary Blues */
.btn-primary {
  background: var(--color-primary-500);
  color: #FFFFFF;
}

.btn-primary:hover {
  background: var(--color-primary-700);
}

/* Semantic Colors */
.alert-warning {
  background: var(--color-warning-50);
  color: var(--color-warning-900);
  border: 1px solid var(--color-warning-500);
}

/* Text Colors */
.text-body {
  color: var(--color-neutral-900);
}

.text-secondary {
  color: var(--color-neutral-600);
}
