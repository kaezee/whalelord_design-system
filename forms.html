Input Fields

Text input components for forms and data entry. Includes multiple states, sizes, and validation styles.

Basic Text Inputs

Standard text input fields in three sizes: small (32px), medium (40px), and large (48px).

React + Tailwind 
// Input Component
const Input = ({ 
  label, 
  size = 'md', 
  placeholder,
  ...props 
}) => {
  const sizes = {
    sm: 'h-8 px-3 text-sm',
    md: 'h-10 px-4 text-base',
    lg: 'h-12 px-5 text-lg',
  };

  return (
    <div className="mb-5">
      {label && (
        <label className="block text-sm font-medium text-pearl-700 mb-2">
          {label}
        </label>
      )}
      <input
        type="text"
        placeholder={placeholder}
        className={`w-full ${sizes[size]} text-pearl-900 bg-white border border-pearl-300 rounded-md focus:outline-none focus:border-kelp-500 focus:ring-3 focus:ring-kelp-500/10 transition-all`}
        {...props}
      />
    </div>
  );
};

// Usage
<Input label="Email Address" placeholder="you@example.com" />
<Input label="Company Name" size="lg" placeholder="Enter company name..." />

Input States

Different visual states for validation and user feedback.

Please enter a valid email address
Looks good!
React + Tailwind 
// Input with Validation Component
const Input = ({ 
  label,
  error,
  success,
  helperText,
  ...props 
}) => {
  const getStateClasses = () => {
    if (error) return 'border-crimson-500 focus:border-crimson-500 focus:ring-crimson-500/10';
    if (success) return 'border-coral-500 focus:border-coral-500 focus:ring-coral-500/10';
    return 'border-pearl-300 focus:border-kelp-500 focus:ring-kelp-500/10';
  };

  const getHelperColor = () => {
    if (error) return 'text-crimson-500';
    if (success) return 'text-coral-500';
    return 'text-pearl-600';
  };

  return (
    <div className="mb-5">
      {label && (
        <label className="block text-sm font-medium text-pearl-700 mb-2">
          {label}
        </label>
      )}
      <input
        className={`w-full h-10 px-4 text-base text-pearl-900 bg-white border rounded-md focus:outline-none focus:ring-3 transition-all disabled:bg-pearl-100 disabled:text-pearl-500 disabled:cursor-not-allowed ${getStateClasses()}`}
        {...props}
      />
      {helperText && (
        <span className={`block text-xs mt-2 ${getHelperColor()}`}>
          {helperText}
        </span>
      )}
    </div>
  );
};

// Usage
<Input label="Email" error helperText="Invalid email address" />
<Input label="Username" success helperText="Username available!" />
<Input label="Read Only" disabled value="Cannot edit" />

Textarea

Multi-line text input for longer content.

Maximum 500 characters
React + Tailwind 
// Textarea Component
const Textarea = ({ 
  label,
  helperText,
  rows = 4,
  ...props 
}) => (
  <div className="mb-5">
    {label && (
      <label className="block text-sm font-medium text-pearl-700 mb-2">
        {label}
      </label>
    )}
    <textarea
      rows={rows}
      className="w-full px-4 py-3 text-base text-pearl-900 bg-white border border-pearl-300 rounded-md focus:outline-none focus:border-kelp-500 focus:ring-3 focus:ring-kelp-500/10 transition-all resize-vertical"
      {...props}
    />
    {helperText && (
      <span className="block text-xs text-pearl-600 mt-2">
        {helperText}
      </span>
    )}
  </div>
);

// Usage
<Textarea 
  label="Message" 
  placeholder="Enter your message..."
  helperText="Maximum 500 characters"
  rows={4}
/>

Usage Guidelines

✅ Do

  • Use clear, descriptive labels for all input fields
  • Provide helpful placeholder text as examples
  • Show validation feedback immediately when possible
  • Use appropriate input types (email, tel, number, etc.)
  • Maintain consistent spacing between form fields
  • Use helper text to clarify requirements

❌ Don't

  • Don't use placeholder text as a substitute for labels
  • Don't make inputs unnecessarily small on mobile
  • Don't show error states before user interaction
  • Don't use vague error messages like "Invalid input"
  • Don't disable inputs without explanation
  • Don't make required fields unclear

Accessibility

✓

Labels: All inputs have associated labels for screen readers

✓

Focus Indicators: Clear focus rings for keyboard navigation

✓

Error Messages: Validation errors are announced to screen readers

✓

Touch Targets: Minimum 44x44px tap targets on mobile
