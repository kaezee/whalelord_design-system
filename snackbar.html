Snackbar / Toast

Brief notification messages that appear temporarily to provide feedback on user actions.

Snackbar Variants

Different variants for various message types: default, success, error, warning, and info.

React + Tailwind 
import { X, CheckCircle, XCircle, AlertTriangle, Info } from 'lucide-react';

const Snackbar = ({ 
  variant = 'default', 
  message, 
  onClose,
  autoHideDuration = 5000 
}) => {
  const variants = {
    default: {
      bg: 'bg-pearl-900',
      text: 'text-white',
      icon: null,
    },
    success: {
      bg: 'bg-coral-500',
      text: 'text-white',
      icon: CheckCircle,
    },
    error: {
      bg: 'bg-crimson-500',
      text: 'text-white',
      icon: XCircle,
    },
    warning: {
      bg: 'bg-amber-500',
      text: 'text-pearl-900',
      icon: AlertTriangle,
    },
    info: {
      bg: 'bg-tide-500',
      text: 'text-white',
      icon: Info,
    },
  };

  const { bg, text, icon: Icon } = variants[variant];

  useEffect(() => {
    if (autoHideDuration) {
      const timer = setTimeout(onClose, autoHideDuration);
      return () => clearTimeout(timer);
    }
  }, [autoHideDuration, onClose]);

  return (
    <div 
      className={`min-w-[320px] max-w-[600px] px-5 py-4 ${bg} ${text} rounded-md shadow-lg flex items-center gap-3 animate-slide-up`}
    >
      {Icon && <Icon className="w-5 h-5 flex-shrink-0" />}
      <span className="text-sm flex-1">{message}</span>
      <button 
        onClick={onClose}
        className={`${text} opacity-80 hover:opacity-100 p-1 transition-opacity`}
      >
        <X className="w-4 h-4" />
      </button>
    </div>
  );
};

// Usage with a Snackbar Provider/Context
import { createContext, useContext, useState } from 'react';

const SnackbarContext = createContext();

export const SnackbarProvider = ({ children }) => {
  const [snackbar, setSnackbar] = useState(null);

  const showSnackbar = (message, variant = 'default') => {
    setSnackbar({ message, variant, id: Date.now() });
  };

  return (
    <SnackbarContext.Provider value={{ showSnackbar }}>
      {children}
      {snackbar && (
        <div className="fixed bottom-6 left-1/2 transform -translate-x-1/2 z-50">
          <Snackbar
            message={snackbar.message}
            variant={snackbar.variant}
            onClose={() => setSnackbar(null)}
          />
        </div>
      )}
    </SnackbarContext.Provider>
  );
};

export const useSnackbar = () => useContext(SnackbarContext);

// In your components
const { showSnackbar } = useSnackbar();
showSnackbar('Order placed successfully', 'success');

Snackbar with Action

Snackbars can include an action button for immediate interaction.

const SnackbarWithAction = ({ 
  message, 
  actionLabel,
  onAction,
  onClose,
  variant = 'default' 
}) => {
  return (
    <div className="min-w-[320px] max-w-[600px] px-5 py-4 bg-pearl-900 text-white rounded-md shadow-lg flex items-center gap-4">
      <span className="text-sm flex-1">{message}</span>
      {actionLabel && (
        <button
          onClick={onAction}
          className="text-kelp-300 hover:text-kelp-200 text-sm font-medium transition-colors"
        >
          {actionLabel.toUpperCase()}
        </button>
      )}
      <button 
        onClick={onClose}
        className="text-white opacity-80 hover:opacity-100 p-1 transition-opacity"
      >
        <X className="w-4 h-4" />
      </button>
    </div>
  );
};

Usage Guidelines

✅ Do

  • Keep messages brief and actionable (under 100 characters)
  • Auto-dismiss after 4-6 seconds (except errors)
  • Show only one snackbar at a time
  • Use success variant for completed actions
  • Provide clear action buttons when applicable
  • Position at bottom center or bottom left

❌ Don't

  • Don't stack multiple snackbars
  • Don't use for critical system errors (use modals)
  • Don't auto-dismiss error messages
  • Don't include long paragraphs of text
  • Don't block important UI elements
  • Don't use for permission requests or confirmations

Accessibility

✓

ARIA Role: Use role="alert" for important messages, role="status" for less critical updates

✓

Screen Readers: Snackbars are announced automatically with proper ARIA attributes

✓

Focus Management: Don't steal focus unless containing interactive elements

✓

Keyboard Dismiss: Allow Escape key to dismiss snackbar
